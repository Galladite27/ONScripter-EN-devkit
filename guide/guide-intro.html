Galladite's ONScripter-EN guide - general introduction

Hello and welcome to my guide to ONScripter-EN! This guide is a major revision (more like a recreation) of my original guide from 2023.

This time around, it is broken down into three sections: firstly, a general quick-start guide, which aims to be a better version of what the old guide was, allowing anybody to create a basic novel game in a short period of time and build his skills from there; secondly, a translation-focused quick-start guide, with information specifically tailored towards translating NScripter games with ONScripter-EN, in even less technical detail than the general quick-start guide, to allow translators to start with the basics as soon as possible; and finally, thanks to my experience using and maintaining the engine over the past two years as well as working on new ports and translations with Eien, a full guide which goes into more detail into all of the main functions of the engine for more serious developers or curious individuals.

In this introduction, I will cover some of the very basics of simply using the engine, which the several guides may expand upon as required. Please expect a warmer and more tailored introduction within the relevant guide to you after reading this page.

Throughout this guide as a whole, you may see various information boxes. They may be as follows: code, images, quotes, tips, or warnings. I hope these distinct sections help to add clarity and interest compared to the old guide.

Use cases

ONScripter-EN is an engine with a lot of history. It is an English-language fork of Ogapee's ONScripter, in turn an open-source port of the very popular Japanese novel game engine NScripter, initally released in 1999. With any software as old as that, there are bound to be very good reasons for its continued usage, but also other reasons for its abandonment in favour of more "modern" solutions.

In-brief:

What ONScripter-EN is good for:

  • Running, playing, and translating games originally made for NScripter on modern systems with ease
  • Somehow, a very good and consistent level of documentation and support
  • A distinctive visual style, even when customised by the user, like that of Tsukihime and Higurashi
  • A very simple and easy to learn (BASIC-like), yet powerful, scripting language
  • Portability to various operating systems
  • A small but dedicated community
  • Years of history, and yet modern-ish features!

What ONScripter-EN is not good for:

  • Technically advanced games (e.g. 3d graphics, advanced animations, integration with external libraries)
  • Less overall available support than larger engines like Ren'Py, or paid solutions
  • Certain limitations in the scripting language mean for less absolute flexibility and extensibility
  • If you want the engine to look a certain way (besides the default), you can't just "download a theme" - you have to make it yourself!

For more information about the history of ONScripter-EN, please see the about page. In terms of its future, as of the time of writing (2025) it is still being actively maintained and supported after a period of inactivity from 2011-2023, with no plans to stop. Since 2023, we have implemented various modernisation features such as widescreen & large resolution support, UTF-8, proportional fonts, and more, while being careful to leave the older functionality of the engine entirely intact and isolated for backwards compatibility purposes, since ONScripter-EN is not only a tool for making new games with but also a compatibility tool for running very old and/or Japanese NScripter-based games.

Fundamentals of using the engine

There are some game files which are ordinarily required for a game to run, and some which are optional. Please see the following table for a non-exhaustive overview of the most common of these files and their purposes.

Name Purpose Required?
0.txt Contains the game "script," or the code for what the game does and when Yes
nscript.dat Obfuscated game script; alternative to 0.txt Yes (if no 0.txt)
default.ttf Font file Yes
arc.nsa Obfuscated game assets archive (CG, audio, &c.); alternative to loose game assets No
ons.cfg Hard-coded engine startup options No
game.id Specifies the game ID - primarily for setting save location No

When the engine is run, briefly, it (1) reads startup options from ons.cfg (if it exists), (2) finds the script and interprets any setup instructions, (3) checks for game assets, game.id, and other such files, and then (4) begins interpretation of the script.

Therefore, the minimum required files for a game to run are the engine, a script, and a font.

Both of the quick-start guides provide a fully set-up and ready to use project which can be immediately be toyed around with alongside the tutorial to build experience and knowledge. The usage of these files in the same quick-start guides will only be covered as necessary for simple VN-creation and translation projects - however, a comprehensive coverage will be given in the full guide.
